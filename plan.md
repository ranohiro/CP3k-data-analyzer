【私の仕事】
臨床検査薬メーカーで、検査試薬の開発を行っている研究開発員です。
試薬は、自動分析装置で血液検体と混合され、単位時間(min.)当たりの吸光度(Abs.)の変化を定量化することで、検査値を返すのが基本的な仕組みになっています。
その検査値結果を解析し、試薬の処方や製法、さらには分析装置のパラメーターを検討することが、私のメインの仕事です。
しかし現行のデータ解析方法に課題があり、アプリ開発の実施を行います。

【開発の目的】
1. CSVデータの可読性改善
2. 吸光度プロファイルの可視化
3. 検量線の構築と可視化
4. 任意区間選択による処理値（傾き）の再計算
5. 再計算値を用いた濃度の再算出
6. データ処理の標準化

【実際の検体測定業務フロー】
測定は以下の4ステップで行われる：
1. キャリブレーター測定
   → 各濃度に対応する吸光度データ取得
2. 検量線作成
   → 吸光度 ↔ 濃度の関係を構築
   → タイプ：折れ線 / スプラインなど
   → ポイント数は試薬依存（例：6点）
3. 一般検体測定
   → 吸光度取得
4. 検量線により濃度算出
5. 検体測定値の解析、試薬開発へのフィードバック

【機能要件】
- StreamlitベースのGUI
- CSV読み込み
- 上段・下段データの自動分離
- プロファイルデータの展開
- キャリブレーションデータの自動抽出
- 検量線の生成（折れ線およびスプライン対応）
- タイムコースデータ表示(生データ表示または起点とする時間を選択して表示)
- タイムコースの任意2点選択による傾き計算
- 傾きから吸光度を算出
- 検量線を用いた濃度再計算
- 複数サンプル比較
- 計算過程の表示（ΔAbs, ΔTime）
(260526追記)
1. キャリブレーター識別ルールを設定可能にし(ハードコードしない)、別途濃度情報をユーザーにより入力可能にする
2. 吸光度の定義を選択可能にする(「区間傾き⊿mAbs/min.、特定時点のｍAbs.、選択区間の平均ｍAbs」をUIから変更可能にする)
3. 区間選択機能を2方式で実装する
   - ポイント番号指定（例：10〜20
   - グラフ上でのクリック選択（2点）
4. 検量線タイプを、UIでユーザーが選択可能にする
   - Piecewise Linear
   - Cubic Spline
5. 内部データ構造を統一する（DataFrame設計、後述）

【出力形式】
1. アーキテクチャ説明
2. データ処理手順
3. 完全なPythonコード（コピペで動作）
4. 実行方法

【仕事の進め方】
まずはstreamlitで動くアプリを試作 →ロジック固定 →PySideでリッチUI化を行う
! streamlit開発も、ステップ分割して段階的な作成を実施。
Step 1：CSVパーサ+データ構造の整形
Step 2：プロファイル可視化
Step 3：吸光度算出の区間選択+傾き(処理値)計算
Step 4：キャリブレーター抽出+検量線作成
Step 5：一般検体の濃度再計算

csvデータのファイル構造：
・CP3k-easy-analyses\docs\CP3k_outputs_data.md #ファイル構造のルールなどを開設した.mdファイル
・CP3k-easy-analyses\data\raw-data\260427-28 (全部) F12 CSV_data.csv #例に使用するcsvファイル

内部データ構造は以下の形式に統一すること：
- samples_df（縦持ち）
  columns:
    request_id
    item_name
    value
    flag
    attribute

- profile_df
  columns:
    request_id
    item_name
    time_sec
    abs_mabs

- calibration_df
  columns:
    item_name
    concentration
    abs_value

検量線は複数タイプをサポートし、ユーザーがUIで切り替え可能にすること：
- Piecewise Linear（必須）
- Cubic Spline（必須）
